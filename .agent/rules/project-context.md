# Project Context (Fonte de Verdade)

A pasta `.context/` é a **fonte de verdade** do projeto para layout, protótipos e referências visuais.

## Obrigatório antes de alterar UI

- Consultar os protótipos interativos (`*-PROTOTIPO-INTERATIVO.md`) da página alvo.
- Conferir as imagens de layout desktop e mobile correspondentes.
- Respeitar o `GHOST-DESIGN-SYSTEM.md` para decisões de visual e motion.

## Conteúdo da `.context/`

- `GHOST-DESIGN-SYSTEM.md`: regras visuais, tokens e direção estética.
- `HOME - PROTOTIPO INTERATIVO.md`: fluxo e detalhamento da Home.
- `PORTFOLIO-PROTOTIPO-INTERATIVO.md`: fluxo e detalhamento do Portfólio.
- `SOBRE-PROTOTIPO-INTERATIVO.md`: fluxo e detalhamento do Sobre.
- Imagens `*.jpg` / `*.png`: referência visual do layout (desktop e mobile).
- `ghost.mp4`: referência de motion/ritmo visual.

## Regra prática

Quando houver divergência entre implementação e `.context/`, **priorizar `.context/`**.
